# TuiBox v0.1 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a tested Linux/macOS TUI client that imports subscriptions and safely manages sing-box proxy or TUN sessions through a narrow privileged daemon.

**Architecture:** A shared domain layer feeds subscription adapters, secure state storage, sing-box config generation, and versioned Unix-socket RPC. The unprivileged `tuibox` binary owns UI and user data. The privileged `tuiboxd` binary validates every request, generates configs, and owns the core process.

**Tech Stack:** Go, Cobra, Bubble Tea, Lip Gloss, YAML v3, x/sys/unix, systemd, launchd, GitHub Actions.
